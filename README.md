## Trabajo práctivo grupal (TPG)

Este repositorio corresponde al **trabajo práctivo grupal iterativo** de un _SQUAD_.

### Estructura
```
/docs                    ← documentación del proyecto
/docs/meetings          ← minutas de reuniones y talleres
/docs/requirements      ← análisis y especificación de requisitos
/docs/design            ← diseño técnico del módulo
/images                 ← imágenes y diagramas
/images/diagrams        ← diagramas (Mermaid/PlantUML/imágenes)
/src                    ← código fuente del backend y pruebas
README.md
```

## Cómo empezar

### Minutas de Reuniones

Para crear una nueva minuta:

1. **Copiar el template:** Clonar `docs/meetings/template-meeting-AAAAMMDD-title.md`
2. **Renombrar el archivo:** Usar formato `meeting-YYYYMMDD-titulo-corto.md`
   - Ejemplo: `meeting-20250115-planificacion-sprint1.md`
3. **Completar la información:** Llenar todos los campos del template
4. **Subir al repositorio:** Commitear y pushear los cambios

### Template de Minutas

El template incluye:
- **Información de la reunión:** fecha, hora, lugar, asistentes
- **Agenda previa:** puntos a tratar
- **Detalle de la minuta:** resumen de cada tema
- **Compromisos asumidos:** tabla con responsables y fechas
- **Material complementario:** documentos y detalles adicionales
- **Próximos pasos:** acciones pendientes

### Requisitos

Para trabajar con requisitos:

1. **Visión del módulo:** Usar `docs/template-vision-modulo.md`
2. **Análisis de interesados:** Templates en `docs/requirements/stakeholders/`
3. **Especificación de requisitos:** Templates en `docs/requirements/specifications/`
4. **Modelo de análisis:** Templates en `docs/requirements/analysis/`
5. **Prototipos:** Templates en `docs/requirements/prototypes/`

### Diseño

Para trabajar con diseño:

1. **Diagrama de contenedores:** Template en `docs/design/containers/`
2. **Diagrama de componentes:** Template en `docs/design/components/`
3. **Diagrama de código:** Template en `docs/design/code/`
4. **Diseño de base de datos:** Template en `docs/design/database/`

### Pruebas

Para trabajar con pruebas:

1. **Pruebas unitarias:** Crear en `src/tests/unit/`
2. **Pruebas de aceptación:** Crear en `src/tests/acceptance/`
3. **Feature files Gherkin:** Crear en `src/features/`

### Imágenes y Diagramas

Para trabajar con imágenes y diagramas:

1. **Diagramas Mermaid:** Crear archivos .md con diagramas Mermaid
2. **Diagramas PlantUML:** Crear archivos .puml
3. **Imágenes:** Subir archivos .png, .jpg, .svg a `/images/`
4. **Diagramas específicos:** Guardar en `/images/diagrams/`

**Herramientas recomendadas:**
- **Mermaid** (recomendado para diagramas de flujo, secuencia, etc.)
- **PlantUML** (para diagramas UML)
- **Figma/Draw.io** (para diagramas visuales)
